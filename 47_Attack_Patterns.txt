GROUP A — CREDENTIAL BRUTE FORCE  (patterns 1–12)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Pattern 1 — MIRAI_DEFAULT_CREDS
  Group     : A
  Protocols : telnet, ssh
  Indicator : root/xc3511 or root/vizxv or root/888888 or root/default in first 3 attempts
  Botnet    : Mirai
  MITRE     : T1110.001

Pattern 2 — MIRAI_ADMIN_SWEEP
  Group     : A
  Protocols : telnet, ssh
  Indicator : admin/admin → admin/1234 → admin/password tried sequentially, ≥2 attempts
  Botnet    : Mirai
  MITRE     : T1110.001

Pattern 3 — MOZI_ROUTER_CREDS
  Group     : A
  Protocols : telnet, http
  Indicator : root/root or Admin/Admin on port 23 and 80 simultaneously
  Botnet    : Mozi
  MITRE     : T1110.001

Pattern 4 — HAJIME_SLOW_BRUTE
  Group     : A
  Protocols : ssh
  Indicator : <2 attempts/min, randomised credential order, session dwell >30 s
  Botnet    : Hajime
  MITRE     : T1110.001

Pattern 5 — CONTROL4_TARGETED
  Group     : A
  Protocols : telnet
  Indicator : exclusively root/t0talc0ntr0l4! — single attempt, then moves on
  Botnet    : Control4
  MITRE     : T1110.001

Pattern 6 — GAFGYT_DEFAULT
  Group     : A
  Protocols : telnet
  Indicator : root/888888 or root/default, inter-attempt interval <500 ms
  Botnet    : Gafgyt (Bashlite)
  MITRE     : T1110.001

Pattern 7 — SATORI_HUAWEI
  Group     : A
  Protocols : telnet
  Indicator : root/Zte521 or supervisor username; often targets port 37215 first
  Botnet    : Satori
  MITRE     : T1110.001

Pattern 8 — FBOT_FBXROUTER
  Group     : A
  Protocols : http, telnet
  Indicator : supervisor/zyad1234 or telecomadmin/admintelecom credential combos
  Botnet    : FBot
  MITRE     : T1110.001

Pattern 9 — GENERIC_DICT_FAST
  Group     : A
  Protocols : telnet, ssh
  Indicator : >10 attempts/min, sequential dictionary ordering, no per-IP state
  Botnet    : Generic scanner
  MITRE     : T1110.001

Pattern 10 — CREDENTIAL_STUFFING
  Group     : A
  Protocols : http
  Indicator : POST /login with base64-encoded JSON bodies, User-Agent rotates per request
  Botnet    : Generic
  MITRE     : T1110.001

Pattern 11 — SINGLE_SHOT_DEFAULT
  Group     : A
  Protocols : telnet, ssh, http
  Indicator : exactly 1 attempt per connection, always admin/admin or root/root
  Botnet    : Generic (Shodan/Censys probe)
  MITRE     : T1110.001

Pattern 12 — SUBNET_COORDINATED
  Group     : A
  Protocols : telnet, ssh
  Indicator : multiple source IPs in the same /24 subnet using identical credentials
  Botnet    : Coordinated botnet
  MITRE     : T1110.001

GROUP B — RECONNAISSANCE & SCANNING  (patterns 13–20)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Pattern 13 — PORT_SCAN_SEQUENTIAL
  Group     : B
  Protocols : tcp
  Indicator : connections to ports 22, 23, 80, 1883 within 10 s from the same source IP
  Botnet    : Masscan / Zmap
  MITRE     : T1046

Pattern 14 — BANNER_GRAB_ONLY
  Group     : B
  Protocols : ssh, telnet
  Indicator : connects, reads banner, disconnects without sending any data (banner_only flag)
  Botnet    : Shodan crawler
  MITRE     : T1046

Pattern 15 — HTTP_FINGERPRINT
  Group     : B
  Protocols : http
  Indicator : GET /, HEAD /, GET /favicon.ico in sequence; reads Server response header
  Botnet    : Generic web scanner
  MITRE     : T1046

Pattern 16 — CGI_PROBE
  Group     : B
  Protocols : http
  Indicator : GET requests to /cgi-bin/, /shell, /command, /cmd within a single session
  Botnet    : Generic CVE scanner
  MITRE     : T1203

Pattern 17 — MQTT_TOPIC_ENUM
  Group     : B
  Protocols : mqtt
  Indicator : subscribes to # wildcard or $SYS/# immediately after CONNECT
  Botnet    : Generic MQTT recon
  MITRE     : T1046

Pattern 18 — UPNP_PROBE
  Group     : B
  Protocols : http
  Indicator : GET /description.xml or GET /rootDesc.xml
  Botnet    : Generic UPnP scanner
  MITRE     : T1046

Pattern 19 — MITRE_T1046_NETSCAN
  Group     : B
  Protocols : tcp
  Indicator : SYN packets across >5 ports, no ACK completion, <100 ms between each
  Botnet    : Generic
  MITRE     : T1046

Pattern 20 — SLOW_RECON
  Group     : B
  Protocols : tcp
  Indicator : one connection every 5–60 min from the same IP over a 24-hour window
  Botnet    : APT
  MITRE     : T1046

GROUP D — POST-EXPLOITATION BEHAVIOUR  (patterns 31–39)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Pattern 31 — WGET_DROPPER
  Group     : D
  Protocols : telnet, ssh
  Indicator : wget http:// or curl http:// executed after successful authentication
  Botnet    : Mirai dropper
  MITRE     : T1105

Pattern 32 — BUSYBOX_WGET_CHAIN
  Group     : D
  Protocols : telnet
  Indicator : /bin/busybox wget followed by /bin/busybox SATORI or /bin/busybox ECCHI
  Botnet    : Satori / Ecchi
  MITRE     : T1105

Pattern 33 — CHMOD_EXECUTE
  Group     : D
  Protocols : telnet, ssh
  Indicator : chmod +x on a file (chmod_seen flag) followed immediately by ./ execution
  Botnet    : Generic dropper
  MITRE     : T1059

Pattern 34 — CRONTAB_PERSISTENCE
  Group     : D
  Protocols : telnet, ssh
  Indicator : crontab -e or echo >> /etc/cron* — establishing persistence
  Botnet    : APT
  MITRE     : T1053

Pattern 35 — IPTABLES_MANIPULATION
  Group     : D
  Protocols : telnet, ssh
  Indicator : iptables -F (flush) or iptables -A (add rule) commands
  Botnet    : Generic (botnet firewall bypass)
  MITRE     : T1562

Pattern 36 — CRYPTO_MINER_INSTALL
  Group     : D
  Protocols : telnet, ssh
  Indicator : xmrig, minerd, or mining pool URL (pool.) referenced in command
  Botnet    : Cryptojacker
  MITRE     : T1496

Pattern 37 — C2_CALLBACK_ATTEMPT
  Group     : D
  Protocols : telnet, ssh
  Indicator : nc, ncat, or bash -i >& /dev/tcp/ reverse shell commands
  Botnet    : APT
  MITRE     : T1059

Pattern 38 — SELF_PROPAGATION
  Group     : D
  Protocols : telnet, ssh
  Indicator : for i in loop scanning subnet combined with telnet connection attempts
  Botnet    : Worm / self-replicating
  MITRE     : T1210

Pattern 39 — LOG_WIPE
  Group     : D
  Protocols : telnet, ssh
  Indicator : rm /var/log/, echo > /var/log/messages, or history -c anti-forensic commands
  Botnet    : APT
  MITRE     : T1070

GROUP E — TIMING & PROTOCOL PATTERNS  (patterns 40–47)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Pattern 40 — HAWKES_BURST_A
  Group     : E
  Protocols : any
  Indicator : >20 events/min high-intensity burst followed by silence >30 min
  Detection : Hawkes process — burst score above BURST_A_RATE_THRESH with
              post-burst silence ≥ BURST_A_SILENCE_SECS (1800 s)
  Botnet    : Generic
  MITRE     : T1498

Pattern 41 — HAWKES_BURST_B
  Group     : E
  Protocols : any
  Indicator : moderate sustained rate of 5–15 events/min over >2 hours
  Detection : Hawkes process — rate within [BURST_B_RATE_MIN, BURST_B_RATE_MAX]
              sustained for ≥ BURST_B_DURATION_SECS (7200 s)
  Botnet    : Generic
  MITRE     : T1498

Pattern 42 — HAWKES_PERIODIC
  Group     : E
  Protocols : any
  Indicator : regular inter-arrival time — coefficient of variation (CV) < 0.30
  Detection : Hawkes classifier computes CV = std/mean of inter-arrival intervals
              over a 2-hour rolling window per source IP
  Botnet    : Generic periodic botnet
  MITRE     : T1498

Pattern 43 — DIURNAL_NIGHT
  Group     : E
  Protocols : any
  Indicator : ≥70% of events from a source IP fall between 00:00–06:00 UTC
  Detection : Hawkes classifier checks DIURNAL_NIGHT_FRAC (0.70) against
              fraction of timestamps in the night window
  Botnet    : Generic (night-campaign botnet)
  MITRE     : T1498

Pattern 44 — MULTI_PROTOCOL_CHAIN
  Group     : E
  Protocols : telnet, http, mqtt
  Indicator : same source IP hits Telnet + HTTP + MQTT within a 60 s window
  Botnet    : Generic
  MITRE     : T1046

Pattern 45 — TLS_DOWNGRADE
  Group     : E
  Protocols : ssh, https
  Indicator : SSLv3 or TLS 1.0 connection attempt; weak cipher suites offered
              (RC4-SHA, RC4-MD5, 3DES-SHA, NULL-MD5, EXPORT-RC4-40, TLS_NULL_WITH_NULL_NULL)
  Detection : TLS honeypot parses ClientHello cipher list and matches _isWeakCipher()
  Botnet    : Generic
  MITRE     : T1562

Pattern 46 — MQTT_QOS_ABUSE
  Group     : E
  Protocols : mqtt
  Indicator : rapid QoS 2 PUBLISH messages to exhaust broker state table
  Botnet    : Generic
  MITRE     : T1499

Pattern 47 — ZERO_DAY_ANOMALY
  Group     : E
  Protocols : any
  Indicator : anomaly score above threshold; no pattern 1–46 matched by heuristics
  Detection : backend classifier — Z-score baseline check + DBSCAN clustering
              in anomaly_detector.py; anomaly feature vector attached to event
  Botnet    : Unknown
  MITRE     : T1203

On-device (ESP32 firmware — attack_patterns.h):
  Groups A, B, D — patterns 1–12, 13–20, 31–39
  Runs classify_session() per session inside EventLogger::logEvent()
  Confidence capped at 70 on-device (_conf() clamp)
  Output fields: pattern_id, pattern_name, pattern_group, confidence, botnet_family, mitre_technique

Backend classifier (Python microservice — backend/classifier/):
  Groups C and E — patterns 21–30, 40–47
  Also re-classifies Groups A/B/D with higher confidence (no cap)
  Heuristic rules  → heuristic_rules.py  (Groups A–D deterministic)
  HMM Viterbi      → hmm_classifier.py   (kill-chain sequencing per session_id)
  Hawkes timing    → hawkes_classifier.py (Group E patterns 40–43)
  DBSCAN anomaly   → anomaly_detector.py (pattern 47)
  GeoIP enrichment → geoip_enricher.py   (country, org, hashed IP)
